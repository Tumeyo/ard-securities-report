FIX Протокол — Гүнзгий Техникийн Шинжилгээ

Financial Information eXchange (FIX) протокол нь дэлхийн санхүүгийн захиалгын холбооны де факто стандарт юм. Хэрэгжилтийн өмнө энэхүү протоколын бүх давхаргыг нарийн ойлгох шаардлагатай.

2.1 Түүх ба Тойм
Гарал үүсэл
1992 — FIX-ийн Эхлэл
1992 онд АНУ-ын Fidelity Investments ба Salomon Brothers хооронд хувийн захиалга дамжуулах протоколоор эхэлсэн. 1998 онд олон нийтэд нээлттэй болж, FIX Trading Community (fixtrading.org) удирдаж эхэлсэн.
Өнөөдрийн байдал
Дэлхийн Де Факто Стандарт
NYSE, NASDAQ, LSE, HKEx зэрэг дэлхийн томоохон биржүүд болон Goldman Sachs, Morgan Stanley, JP Morgan зэрэг бүх томоохон брокерууд FIX ашигладаг. Өдөрт хэдэн тэрбум захиалга FIX-ээр дамждаг.
Хэрэглээний хүрээ
Олон Хөрөнгийн Ангилал
Хувьцаа (Equities), Бонд (Fixed Income), Деривативт (Derivatives), Форекс (FX), Криптовалют зэрэгт хэрэглэгддэг. Ард Секюритиз-д US Equities (NYSE/NASDAQ) буюу хувьцааны арилжааны хэсэг хамааралтай.
Хэн удирддаг вэ
FIX Trading Community
fixtrading.org вэбсайтад бүх техникийн баримт бичгүүд, FIX Dictionary, message specifications байдаг. Олон улсын зах зээлд ороход энэ сайтаас шаардлагатай spec-ийг татаж авах боломжтой.

2.5 FIX Session Layer — Холболтын Менежмент

Session layer нь TCP/IP дэх FIX холболтын найдвартай байдлыг хангадаг. Дарааллын дугаар (Sequence Number), алдагдсан мессеж сэргээх, heartbeat зэргийг удирдана.

MsgTypeНэрТайлбарХэн илгээх вэ
ALogonХолболт тогтоох, HeartBtInt тохиролцохInitiator (Ард Сек)
5LogoutХолболтыг журмын дагуу дуусгахХоёр тал
0HeartbeatХолболт амьд байгааг батлахХоёр тал (автомат)
1Test RequestHeartBtInt хэтэрсэн үед TestReqID-тэй Heartbeat шаардахХоёр тал
2Resend RequestАлдагдсан мессежийн дарааллын дугаарыг дахин хүсэхХоёр тал
4Sequence ResetДарааллын тоог дахин тохируулах (GapFill эсвэл Reset)Хоёр тал
Session lifecycle — Ард Сек (Initiator) ↔ PSPL (Acceptor) 
1. TCP/IP холболт: Ард Сек → PSPL IP:Port (TLS шифрлэлттэй)

2. LOGON (Ард Сек → PSPL)
   35=A | 108=30 (HeartBtInt=30сек) | 98=0 (EncryptMethod=None) | 141=Y (ResetOnLogon)

3. LOGON хариу (PSPL → Ард Сек)
   35=A | 108=30  ← Зөвшөөрсөн

4. Арилжаа эхэлнэ: New Order (D), Cancel (F), Cancel/Replace (G) илгээх

5. Heartbeat (хоёр тал, 30 секунд тутамд)
   35=0  ← "Амьд байна"

6. Хэрэв 30+ сек мессеж байхгүй → Test Request (35=1) илгээнэ
   Хариу ирсэн: ок | Хариу ирээгүй: Logout → reconnect

7. LOGOUT
   35=5 | 58=Session ended normally
⚠️
Дарааллын дугаар (Sequence Number) — чухал! FIX session нь хоёр талын илгээсэн болон хүлээн авсан мессежийн дарааллын дугаарыг хянана. Дугаар алдагдсан тохиолдолд Resend Request (35=2) илгээж алдагдсан мессежүүдийг сэргээдэг. FIX engine файлд sequence number-ийг хадгалдаг тул системийн сэргэлтийн дараа ч дарааллаа хадгалсан байна.

OMS / EMS Архитектур ба Полярисын Өргөтгөл

Полярис нь одоогоор бүрэн OMS (Order Management System) үүрэг гүйцэтгэж байна. Олон улсын FIX интеграцийн тулд OMS-ийн доор шинэ FIX adapter давхарга болон pre-trade risk engine нэмэх шаардлагатай.

4.2 Техникийн Интеграцийн Алхамууд
Алхам 1
PSPL-ийн FIX Specification авах
PSPL-аас FIX Connectivity Guide, Supported Messages список, CompID тохиргоо, IP/port мэдээллийг авна. Зөвхөн PSPL-ийн account manager-ийн дамжуулалтаар боломжтой.
Алхам 2
UAT орчин тохиргоо
PSPL-ийн sandbox (UAT) орчинд холболт хийнэ. Test CompID, test IP/port ашиглан CASPO-д хэрэгжүүлсэн FIX adapter-ийн анхны тестийг хийнэ. Logon/Logout, Heartbeat баталгаажуулна.
Алхам 3
FIX Message тестлэлт
UAT дахь test scenarios:
• New Order (Market) → Execution Report (Fill)
• New Order (Limit) → New → PartialFill → Fill
• Order Cancel → Canceled
• Reject scenarios → Business Message Reject (j)
• Reconnect after disconnect
Алхам 4
PSPL Certification
PSPL-ийн техникийн баг UAT-д холбогдож официал certification test хийнэ. Хэрэгжүүлэлт зөв байвал production CompID болон IP/port олгоно.
Алхам 5
Production — Pilot
Ард Секюритиз-ийн дотоод данс ашиглан жинхэнэ захиалга гүйцэтгэнэ. Бүх execution report, settlement дараалал шалгана. Зөвхөн сонгосон хэрэглэгчдэд нэвтрүүлнэ.

[SESSION]

SenderCompID=ARDSEC         # PSPL-оос баталгаажуулсан

SocketConnectHost=fix.pspl.com  # PSPL-аас олгосон IP/hostname
SocketConnectPort=8443         # PSPL-аас олгосон порт

SocketUseSSL=Y            # TLS 1.2+ шаардлагатай
ResetOnLogon=Y

Аюулгүй байдлын шаардлагууд: TLS 1.2+ шифрлэлт заавал хэрэгтэй. PSPL-ийн зарим тохиолдолд IP whitelist шаардах тул Ард Секюритиз-ийн production серверийн static IP-г PSPL-д бүртгүүлэх шаардлагатай. CompID болон нууц үг production дахь environment variable-д хадгалагдана — hard-code хийж болохгүй.
